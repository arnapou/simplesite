Logs

The site logs a number of things such as 404, 500, etc ...

The log folder is in the path_logs config option and if it is not defined we use the logs subfolder of the path_cache option

A retention of several days is kept and configurable with the log_max_files option.

The log level can be chosen with the log_level option which must have one of the following values:

  • debug
  • info
  • notice (par d√©faut)
  • warning
  • error
  • critical
  • alert
  • emergency

Error pages

At this stage, we only consider 40X and 50X.

There are pre-configured pages for this, examples:

A page that crashes to view the default 500 page

A page that does not exist to view a custom 404 page

To customize error pages, you just need to have a templates folder configured via path_templates and define the following twig files in that folder:

  • error.40x.twig
  • error.50x.twig
